#### 게시글 수정 권한 관련
> 게시글 삭제 시에는 비밀번호 입력 후 버튼 클릭과 동시에 삭제 절차를 처리해서 크게 어려움이 없었습니다.
> 그런데 업데이트는 비밀번호 입력 후 한템포 쉬었다가 작업 완료 후 다시 제출을 하는 방식이라 입력한 비밀번호를 어디에 저장 해야할지 몰라 어려움이 있었습니다.

![스크린샷 2024-01-11 오전 11.10.14.png](readMeImages%2F%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7%202024-01-11%20%EC%98%A4%EC%A0%84%2011.10.14.png)
위 이미지는 비밀번호 입력 후 게시글 수정 페이지에 접근이 가능한지 판별해주는 Controller 입니다.
비밀번호 체크 후 일치한다면 `token` 이라는 임시 `접근 허가증`을 부여합니다.

해당 값은 `article`의 `id` 와 동일한 값이며 누구나 임의로 수정할 수 있고 예측 가능한 값이여서 현재는 임시로 적용해둔 상태입니다.

해당 토큰을 가지고 `/article/{id}/modify?token={id}` 주소로 이동하게되면 토큰 검증 절차를 거칩니다.

![스크린샷 2024-01-11 오전 11.13.46.png](readMeImages%2F%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7%202024-01-11%20%EC%98%A4%EC%A0%84%2011.13.46.png)
위 이미지는 게시글을 수정할 화면의 controller 입니다. <br/> 
`@RequestParam("token")` 에너테이션으로 쿼리스트링 `token`의 값을 받아오고 있습니다. <br/>
이후 토큰 값이 비어있는지 체크하고 비어있다면 `accessDenied` 페이지를 표시합니다. <br />
토큰 값이 존재한다면 해당 토큰이 올바른 토큰인지 검증하는 과정을 거칩니다.

```java
boolean isPossible = Long.parseLong(token) == id;
```
현재는 단순히 값이 일치하는지 여부만 판별하고 있습니다. 추후 이 부분을 세션이나 난수값 토큰으로 변경하여 서버에서 관리하는 방식으로 변경할 예정입니다. <br/>

토큰 판별 후 정상처리되었다면 `id` 를 통해 게시글을 읽어옵니다. 이후 게시글을 화면으로 전달해주고 게시글 수정 페이지로 이동합니다. 
<br />

만약 토큰 판별 결과 값이 일치하지 않는다면 `accessDenied` 페이지를 표시합니다.

### 1. 해시태그 기능

- 게시글을 작성할때, 사용자가 입력한 내용을 바탕으로 해시태그를 추출한다.
    - 해시태그는 사용자가 작성한 내용 중, `#` 으로 시작하는 단어를 의미한다.
- 특정 해시태그를 가진 게시글 목록을 볼수 있는 화면이 필요하다.
- 게시글 단일 조회 페이지의 내용 뒤쪽에 사용된 해시태그 목록이 표기된다.
    - 링크로서 동작하며, 클릭시 해당 해시태그가 포함된 게시글 목록을 보는 화면으로 이동된다.
    

### 2. 검색 기능

- 게시글 목록을 확인할 수 있는 페이지에, 검색을 위한 UI가 추가된다.
    - 사용자가 검색어를 입력해서 검색을 할 수 있다.
- 검색을 하면서 검색 기준을 선택할 수 있다.
    - 제목
    - 내용
- 개별 게시판이 선택된 상태론 해당 게시판 내에서, 전체 게시판이 선택된 상태론 전체 게시글 중 검색한다.

### 3. 게시글 추가 기능

- 게시글이 작성된 이후 게시글에 이미지를 추가할 수 있다.
    - 게시글 단일 조회 페이지에서 이미지 추가 버튼을 통해 추가한다. 추가할땐 게시글을 작성하면서 전달한 비밀번호를 같이 전달해야 한다.
    - 추가된 이미지는 제목 이후 내용 이전에 추가된 순서대로 배치된다.
- 개별 이미지를 삭제할수 있다. 삭제할땐 게시글을 작성하면서 전달한 비밀번호를 같이 전달해야 한다.
- 게시글 단일 조회 페이지에 이전글 다음글 기능이 추가된다.
    - 이전글 링크를 클릭하면 같은 게시판의 게시글 중, 현재 게시글보다 나중에 작성된 게시글 중 가장 일직 작성된 게시글 단일 조회 페이지로 이동한다.
    - 다음글 링크를 클릭하면 같은 게시판의 게시글 중, 현재 게시글보다 먼저 작성된 게시글 중 가장 늦게 작성된 게시글 단일 조회 페이지로 이동한다.
    - 전체 게시판에서 게시글을 조회하는 경우, 게시판 구분 없이 동일한 기준으로 동작한다.
</details>

## 문제 발생 및 해결과정

<details>
<summary> 테이블 설계시 게시글과 이미지 연결 문제 </summary>

### 문제상황

**현재 이 문제와 관련된 기능 구현은 하지 않았고 테이블 설계 까지만 해둔 상태 입니다** <br />

도전과제 요구사항을 보면 게시글에 `plain text` 만 사용해야 하고 이미지를 추가할 수 있도록 해라 라는 부분이 있습니다.
보통 웹에서 게시글을 처리할 때 `WYSIWYG Editor` 라는 것을 사용합니다.

이 에디터를 사용하면 `html` 요소를 만들어 이를 string 형태로 변환 후 데이터베이스에 이 내용을 그대로 저장하는 방식을 사용하기도 한다고 합니다.

그런데 요구사항에는 `plain text`만 사용해야 한다고 하기에 이 방식은 사용하면 안되는 것으로 인지하고 아래와 같은 방식으로 처리했습니다. 

### 해결

몇개의 이미지가 들어올지 모르는 상황에서 `article`테이블에 image_url_01, image_url_02 ... 이런식으로 다 추가해둘수는 없기에 `images` 라는 테이블을 새로 만들었습니다. <br/>
이 테이블은 모든 이미지를 저장할 목적을 가지고 있고 `article_id`라는 컬럼을 가지고 있습니다. <br/>

추후 게시글 조회시 `images` 테이블 에서 `article_id`이 일치하는 값만 필터링 해서 가져온 뒤 화면단으로 `images` 라는 값을 넘겨주면 쉽게 해결할 수 있을 것 같다고 생각하고 있습니다.


</details>

<details>
<summary> 특정 테이블이 생성되지 않는 문제 </summary>

### 문제상황

다른 테이블은 모두 정상적으로 생성되는데 특정 테이블만 생성되지 않던 문제

### 해결

`category` 라는 테이블에서 현재 게시판을 관리하고 있습니다.

![스크린샷 2024-01-11 오전 10.42.18.png](readMeImages%2F%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7%202024-01-11%20%EC%98%A4%EC%A0%84%2010.42.18.png)
해당 테이블에는 `sort_order` 라는 컬럼이 있는데 이 값은 이전에 `order` 라는 값으로 생성하고 싶었습니다.

그런데 `order`는 `order by`를 위한 예약어 였고 해당 이름으로는 컬럼을 생성할 수가 없었습니다. <br/>
`sort_order` 라는 컬럼으로 변경하여 해결했습니다.

</details>
